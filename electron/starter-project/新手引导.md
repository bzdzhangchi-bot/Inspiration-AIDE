# Inspiration 新手引导

欢迎。你现在看到的是应用自带的 Starter Project，可以直接拿来熟悉整个工作台。

## 先做这 5 件事

1. 在左侧 Explorer 里点开不同文件，感受多文件切换。
2. 打开 `src/main/java/com/inspiration/starter/StarterWorkspaceApplication.java`，看看 Spring Boot 应用入口。
3. 打开 `src/main/resources/application.properties`，看看常见后端项目里配置文件通常放在哪里。
4. 打开 `examples/preview-demo.html`，看看右侧预览区如何实时显示 HTML。
5. 打开 `examples/mermaid-demo.md`，体验 Markdown 和 Mermaid 图表预览。
6. 展开底部 Terminal，执行一个简单命令，比如 `pwd` 或 `ls`。
7. 去 `Settings` 页配置你要连接的模型提供方，再回到 Chat 开始提问。

## 这个应用主要怎么用

### 1. Project

Project 页是你日常停留最多的地方，主要包含三块：

- Explorer：浏览项目、创建文件、重命名、复制、删除
- Editor：直接编辑文本文件
- Preview：预览 Markdown、HTML、PDF 等内容

如果你只是想快速看一个项目、改几个文件、顺手跑命令，这一页就够用了。

### 2. Chat

Chat 区适合做这些事：

- 让模型解释当前文件
- 让模型帮你制定修改方案
- 让模型生成补丁或命令
- 结合当前工作区上下文继续迭代

建议提问方式尽量具体，例如：

> 帮我总结当前项目结构，并告诉我应该先看哪几个文件。

或者：

> 基于当前打开的文件，帮我列一个最小可行改动方案。

### 3. Terminal

Terminal 适合做真正要落地执行的事，比如：

- 查看目录结构
- 运行构建和测试
- 执行 git 命令
- 跑本地开发服务

建议把“聊天里讨论方案”和“终端里执行动作”分开，这样排查问题会更清楚。

### 4. Git

Git 页适合查看：

- 当前仓库是否干净
- 哪些文件已修改
- diff 内容
- commit 历史

如果你不想在命令行里来回敲 git，这里会更直观。

### 5. Settings

第一次使用，最重要的是先在 `Settings` 里确认：

- Provider
- Base URL
- API Key
- Model
- Interaction Mode

没配好这些，Chat 和 Agent 能力通常不会正常工作。

## 推荐的第一次上手路径

### 路径 A：只熟悉界面

1. 打开几个文件
2. 看看右侧预览
3. 试试 Terminal
4. 切到 Git 页看看状态

### 路径 B：完整跑一遍工作流

1. 在 `Settings` 里填好模型配置
2. 回到 Chat，问一句“这个 Starter Project 是干什么的？”
3. 让助手帮你修改 `README.md`
4. 在 Project 里检查改动
5. 在 Git 页查看 diff

## 下一步建议

当你已经熟悉界面后，可以：

- 用左上或侧边入口打开你自己的项目目录
- 把这个 Starter Project 当作草稿区继续保留
- 在自己的真实仓库里重复一遍刚才的工作流

如果你想验证预览能力，继续打开下面两个文件：

- `examples/preview-demo.html`
- `examples/mermaid-demo.md`

如果你想顺手体验 Java 项目内容，也可以继续看：

- `pom.xml`
- `mvnw`
- `src/main/java/com/inspiration/starter/StarterWorkspaceApplication.java`
- `src/main/java/com/inspiration/starter/StarterWorkspaceController.java`
- `src/main/java/com/inspiration/starter/StarterWorkspaceService.java`
- `src/main/resources/application.properties`
- `src/test/java/com/inspiration/starter/HelloInspirationTest.java`
